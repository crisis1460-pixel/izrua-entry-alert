# 엔트리 터치 알림 봇 — 기획서 (v3, 확정)

2026-07-22 기획 세션 완료. 자동매매 컨셉은 폐기(투입원가 대비 수익률 불량 판단),
"TradingView 아이디어 기반 엔트리 터치 알림 봇"으로 전환. **PC 상시가동 불가 제약이
확인되어 v1의 로컬 실행 구조를 GitHub Actions 서버리스로 변경(v2 핵심 수정).**

## 확정 결정 전체

| # | 항목 | 결정 |
|---|---|---|
| 1 | 기존 자동매매 봇 | 중단(정지 확인됨) + 기록 보존 (cache/, logs/, trade_journal) |
| 2 | 실행 환경 | **새 공개(public) GitHub 레포 + Actions 서버리스** — 공개 레포는 Actions 무료 무제한. cron-job.org(워쳐에서 검증된 방식)가 트리거. 워쳐 레포는 무수정 유지 |
| 3 | 시총 top-200 소스 | **CoinGecko** `/coins/markets` 1콜 (Demo 키, 일 1회) → 업비트 KRW 교집합 = 유니버스(~100개 예상) |
| 4 | 알림 트리거 | 위에서 하락 → entry +1% 이내 진입 시 **예고 1회** → entry 터치/하향돌파 시 **본알림 1회** (하방만). 한 체크에서 동시 감지되면 본알림만 발송 |
| 5 | 재알림/만료 | 글당 본알림 1회 후 완료(재알림 없음), 미터치 글은 게시 후 **7일 만료** |
| 6 | 수집 주기 | **4시간마다** (워쳐와 동일 리듬, 심볼당 sleep 3초). 첫 실행이 max_post_age=168h로 "1주일치 백필"을 자연 커버 |
| 7 | 알림 필터 | 수집은 entry 있는 글 **전부 저장**, 알림은 **등급 C 이상 + 코인당 3건/일**(점수순)만. 전부 config 조정 가능 |
| 8 | 본알림 양식 | 표준형 5줄 (아래 목업). 예고알림은 같은 틀에서 헤더만 `⚠️ [접근]`으로 |
| 9 | 가격 체크 주기 | **5~10분마다** cron-job.org 트리거. 현재가 + 직전 체크 이후 **1분봉 저가**를 되짚어 판정 → 스파이크 터치도 놓치지 않음(알림만 최대 체크 주기만큼 지연) |
| 10 | 기존 코드 정리 | 재사용 로직(등급/R:R/환산/notifier/extractor) 이식 완료 판단 후 **완전 삭제** |
| 11 | 알림 부가정보 | **작성자 적중률·건수·화이트리스트 표시**(워쳐 DB 아티팩트의 chartist_stats/whitelist 읽기 — watcher_feed.py 코드 이식. 워쳐 미추적 작성자는 `기록없음`) + **시총 순위·등급 표기**: 💎 1~10위 · 🥇 11~50위 · 🥈 51~100위 · 🥉 101~200위 (경계 config 조정 가능) |
| 12 | 유사 레벨 병합 | 같은 코인에서 엔트리가 서로 **±1% 이내인 글들만 한 클러스터로 병합** — 클러스터 상단 엔트리 터치 시 1회 본알림(엔트리 존 표기), 출처 최신순 나열. ±1% 밖이면 별개 레벨로 각각 알림. 클러스터 밴드 config 조정 가능 |
| 13 | 아침 요약 | **미채택** (사용자 결정). 봇 사망 감지는 cron-job.org의 실행 실패 알림(무료 기본기능)으로 갈음 |

### 본알림 목업 (v3 확정)
```
🎯 [터치] LINK 🥈 시총 14위 · ₩12,050
엔트리 존 12,050~12,100 하향 돌파
🏁 TP 13,900 (+14.9%) · 🛑 SL 11,300 (-6.6%) · R:R 1:2.3
✍️ CryptoXYZ · 적중 67% (12건) ✅화이트 · 글 2일 전 · B등급
📎 출처(최신순)
 · 2일 전 CryptoXYZ — tradingview.com/…/AbC123
 · 5일 전 TraderKim — tradingview.com/…/XyZ789
```
- 예고알림은 같은 틀, 헤더만 `⚠️ [접근]`
- 링크를 크롬으로 열려면(봇이 강제 불가): 텔레그램 설정 → 데이터 및 저장공간 → 인앱 브라우저 끄기

## 아키텍처 (v2 — 서버리스)

```
[새 공개 레포 (이름 미정) — GitHub Actions]
│
├─ 수집 잡 (4h, cron-job.org 트리거)
│    CoinGecko top-200(1콜, 일 1회 캐시) ∩ 업비트 KRW → 유니버스
│    → TradingView 아이디어 수집 (tradingview-scraper 계열 + 워쳐 extractor 이식)
│    → entry 있는 글 전부 SQLite 저장 (SL/TP/작성자/팔로워/글나이/URL 포함)
│    → DB를 Actions 아티팩트로 저장 (워쳐와 동일 패턴)
│
└─ 가격체크 잡 (5~10분, cron-job.org 트리거)
     아티팩트 DB 로드 → 활성 레벨별 Upbit REST 현재가(50심볼=1콜)
     + 직전 체크 이후 1분봉 저가 조회 → 예고/터치 판정
     → 텔레그램 발송 (기존 upbit_izrua 봇 토큰, GitHub Secrets)
     → 상태 갱신(watching/previewed/touched/expired) 후 아티팩트 재저장
```

- 텔레그램 봇은 기존 것 재활용 (TELEGRAM_BOT_TOKEN/CHAT_ID를 새 레포 Secrets로)
- 로컬 PC·서버 완전 불필요, 비용 0원 (공개 레포 Actions 무료 무제한 + cron-job.org 무료)
- 유의: 공개 레포이므로 코드·아티팩트(수집 DB)가 외부에서 열람 가능 (토큰은 Secrets라 안전)

## GitHub 무료한도 검증 (2026-07-22, 공식 문서 확인 완료)

- 예상 사용량: 수집 6회/일×15분(2,700분/월) + 가격체크 144~288회/일×~1분(4,300~8,600분/월)
  = **월 7,000~11,300분 상당**
- 공개 레포는 표준 러너 실행시간 **완전 무료**(공식 문서 원문 확인) + **아티팩트 스토리지도
  500MB 한도에 미집계**(GitHub 커뮤니티 공식 답변 확인) → 한도 초과 원천 불가
- 워쳐(비공개)의 월 2,000분 한도와 완전 별개 (한도는 비공개 레포 사용분에만 적용)
- **주의: 비공개로 만들면 즉시 한도 초과** (월 $55+ 과금) — 공개 레포가 무료 운영의 필수 조건
- 기타: PAT API 5,000회/h(사용 ~12회/h), 동시 잡 20개(사용 1~2개) — 전부 여유

## 리서치 근거 (2026-07-22 웹 조사)

- 동일 컨셉 공개 프로젝트 없음. 수집은 `tradingview-scraper`(PyPI) 선례, 파싱은 워쳐
  extractor가 사실상 유일 선행 사례. TradingView는 Cloudflare 차단(1020 밴) 있어
  저빈도+백오프+쿠키 필수.
- Upbit REST 초당 10회(월캡 없음) — 5분 체크는 한도의 0.03% 수준. CoinGecko Demo
  월 10,000콜 중 ~30콜 사용. CMC는 폴백 후보.
- 알림봇 관례: 레벨당 1회성 트리거, 접근 밴드, 추출 숫자의 현재가 대비 sanity check,
  entry 배열(존/범위) 대응.

## 남은 작업 (개발 착수 시)

1. 새 레포 이름 결정 + 공개 레포 생성 + Secrets 등록 (텔레그램/CoinGecko/PAT)
2. cron-job.org에 잡 2개 추가 등록 (수집 4h / 가격체크 5~10분)
3. 워쳐 extractor·등급 로직 이식 → 수집 잡 구현 → 가격체크 잡 구현 → 라이브 검증
4. 검증 완료 후 로컬 upbit_bot 자동매매 코드 완전 삭제 (결정 #10)
