# 자체 적중 DB — 기획서 (v1, 리서치 기반)

2026-07-23 웹 리서치 3건(상용 플랫폼 / 퀀트 방법론 / 오픈소스 구현 관례) 종합.
목표: 워쳐 적중률 의존에서 점진 졸업 — 우리 봇 고유의 "진입가 터치 시점부터"의
성과 데이터를 축적해 작성자 신뢰도를 자체 산출한다.

## 왜 우리 방식이 워쳐보다 정직한가

워쳐는 "글 알림 후 TP/SL 중 뭐가 먼저 닿았나"를 재지만 **진입 도달 여부를 안 본다**
(진입 기회가 없었어도 hit). 우리는 터치 순간을 정확히 알고 5분(현 2분) 단위로 감시
중이므로 **"실제 진입 가능했던 시점부터"** 판정할 수 있다 — 실전에 더 가까운 지표.

단, 리서치가 경고한 함정: 터치 조건부 평가는 승률이 체계적으로 낮게 측정된다
(터치 자체가 단기 역행 신호 — 지정가 adverse selection 문헌). 따라서 워쳐 수치와
직접 비교하지 말고, **터치율(글 중 진입 도달 비율)을 별도 지표로 항상 병기**한다.

## 판정 엔진 (오픈소스 관례 기반)

| 상황 | 판정 | 근거 |
|---|---|---|
| 터치 후 TP1 도달 | **hit** | 업계 표준(Cornix 계열 트래커 관행) |
| 터치 후 SL 도달 | **miss** | 〃 |
| 같은 평가구간에 TP·SL 동시 | **miss** + `ambiguous=1` 플래그 | freqtrade 명문화("자본 보호 우선"). 플래그는 추후 하위 타임프레임 재검사 업그레이드용 |
| TP 없음 | **타임박스**: 터치 후 7일 수익률 부호로 판정 (`judgment_mode='timeboxed'`, 48h 수익률도 참고 기록) | altfins-audit 등 감사 프로젝트 표준. SL 없는 글 76% 실측 — 표본 확보에 필수 |
| SL 없음 | TP1 로 승패 판정, R 계산만 제외 | ATR 합성 SL은 저자 의도 왜곡이라 비권장(선례 약함) |
| 미터치 7일 만료 | `expired` — 승패 어디에도 미포함, **터치율 분모에만 포함** | 선택편향 처방 |
| 부가 기록 | `best_tp_hit`(최고 도달 TP), R-멀티플, 터치 후 24h/72h 수익률 | 나중에 지표 확장용 원천 데이터 |

평가 비용: 이미 2분마다 해당 코인 시세를 조회 중이라 **추가 API 호출 사실상 0**.

## 지표 체계 (퀀트 리서치 권장 조합)

1. **R-멀티플**: `R = (청산가−진입가)/(진입가−SL)`, [−1, +5] 클리핑(윈저라이즈)
2. **최신성 가중**: `w = 0.5^(경과일/90)` (반감기 90일), 유효표본 `n_eff = (Σw)²/Σw²`
3. **승률 축**: 워쳐 DB를 사전확률로 쓰는 베이지안 수축 —
   `Beta(m·p_watcher+1, m·(1−p_watcher)+1)`, 사전강도 m=10.
   자체 표본이 쌓일수록 워쳐 영향이 자연 감소 → **별도 전환 스위치 없이 자동 졸업**
4. **랭킹 주지표**: 보수적 기대값 `E_LB = mean_w(R) − 1.28·std_w(R)/√n_eff`
   (소표본이면 자동으로 낮아져 신인 과대평가 방지 — "2승0패 > 60승40패" 문제 해결)
5. **노출 규칙**: n_eff < 5 "표본 부족" / n_eff ≥ 30 등급 부여 (TipRanks·ZuluTrade 사상)
6. E_LB 지속 음수 작성자 = 역신호 후보 태깅 (Finfluencers 연구: 56%가 anti-skilled)

## 안티게이밍 (상용 플랫폼 관행)
